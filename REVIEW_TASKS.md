# Предложенные задачи по улучшению кодовой базы

## 1) Исправление опечатки
**Задача:** переименовать файл `ttreadme.md` в `README.md` (или объединить с существующим `readme.md`) и исправить ссылки на него.

**Почему:** текущее имя `ttreadme.md` выглядит как опечатка и ухудшает навигацию по документации.

---

## 2) Исправление ошибки в коде ✅ выполнено
**Статус:** исправлено в `scenario_simulator`: для `adjust_production` / `adjust_consumption` отправляется query-параметр `amount`.

**Почему:** это приводит к несоответствию контрактов API и падению сценарного шага с `422 Unprocessable Entity` при попытке симуляции этих действий.

---

## 3) Исправление комментария/документации ✅ выполнено
**Статус:** `readme.md` синхронизирован с фактическим составом сервисов из `docker-compose.yml`.

**Почему:** в текущей схеме упомянуты сервисы (`stream-joiner`, `feature-store`, `training_service`, `dependency-graph-service`, `impact-propagation-service`, `loss-aggregator`, `metadata_service`, `external_data`), которых нет в compose-конфигурации. Это вводит в заблуждение при запуске и поддержке стенда.

---

## 4) Улучшение теста
**Задача:** добавить интеграционный тест в `scenario_simulator`, который проверяет контракт вызова отраслевых сервисов для `adjust_production` и `adjust_consumption` (включая имя параметра `amount`).

**Почему:** сейчас регресс в имени параметра не покрыт тестами; такой тест предотвратит повторное нарушение межсервисного контракта.
